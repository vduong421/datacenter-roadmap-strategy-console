# Roadmap Analysis Skill

## When Used

Use when the user asks about roadmap priority, demand, risk, readiness, competitive gap, or quarterly review planning.

## Input

- ranked initiatives
- deterministic reasoning fields
- customer segment count
- competitor count
- quarterly review message

## Expected Output

- concise answer
- evidence from scores
- next action
- recommendation
- decision
- risks
- operator actions

## Rules

- Rank initiatives using deterministic scores only.
- Highlight high-risk initiatives separately from high-priority initiatives.
- Explain why an initiative should be committed, prototyped, or monitored.
